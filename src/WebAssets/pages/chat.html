❓ Mendix GPT Extension Help

🚀 Getting Started

  1. Open Mendix GPT Extension from the Extensions menu in Studio Pro.
  2. Click the ⚙ gear icon to open Settings.
  3. Select a provider and enter your Anthropic or OpenAI API key. It is encrypted locally and never shared.
  4. Click the ↻ refresh button to load your App Context.
  5. Start chatting!

💬 Chat Modes

Select from the dropdown next to the input:

  • Agent — The selected AI provider can read your app model and create or edit microflows. Use for making changes.
  • Ask — Read-only mode. The selected AI provider inspects your model but cannot modify anything. Use for questions and analysis.

📄 Active Document Tracking

AIDE automatically detects which document you have open in Studio Pro and shows it in a bar below the context status.

When you send a message, this is automatically included as context so the selected AI provider knows what you’re looking at. Click the + button on the indicator bar to add it as an explicit @-reference chip.

If you add explicit @-chips, the automatic active document context is skipped to avoid redundancy.

🖱 Right-Click Context Menu

Right-click any document (microflow, page) in the App Explorer or any entity in the Domain Model editor:

  • Explain with Mendix GPT Extension — Opens the extension and immediately sends “Explain @Module.Element” to the selected AI provider.
  • Add to Mendix GPT Extension Context — Opens the extension and adds an @-reference chip. You type your own question.

Chips are color-coded: Entity Microflow Page

📎 Attachments

Click the ➕ button, drag & drop onto the input area, or paste from clipboard:

Images (JPEG, PNG, GIF, WebP — max 20 MB each):
Shown as thumbnail previews. The selected AI provider can analyze the image.

Text files (max 500 KB each):
Shown as file chips. Content is sent as formatted code blocks.

Supported text types:

  • Code: .java .js .css .html .xml
  • Config/Data: .json .yaml .yml .properties .csv .sql
  • Documentation: .md .txt .log

Click the × on any thumbnail or chip to remove it before sending.

💬 Chat History

Conversations are auto-saved. Access them via the 📋 History button. Click a conversation to load it. Click × to delete.

Enable Auto-Load Last Conversation in Settings to restore your most recent chat on startup.

📤 Export

Click the ⬇ Export button to download the current conversation as a Markdown file. Optionally include tool activity between messages.

🔧 What The AI Can Do

Read (9 tools): List modules, entities, associations, enumerations, pages, microflows. Get full details on entities and microflows. Search any element by name.

Write (5 tools): Create microflows (21 activity types), rename microflows, add activities, replace microflows, edit activity properties.

⚠ Known Limitations

  • No decision, loop, or merge creation (sequential activities only — the AI guides manual placement)
  • No page or nanoflow creation
  • No entity or enumeration creation (read-only domain model)
  • No security rule APIs

Data Processing Consent

Before using Mendix GPT Extension, please understand how your data is processed:

  • App model data (entity names, attributes, microflow structures, page names) is sent to the selected provider to provide context for AI responses.
  • Your chat messages are sent to the selected provider for processing and response generation.
  • Anthropic or OpenAI may process your data depending on the configured provider.
  • Conversation history is stored locally on your machine with DPAPI encryption. Up to 50 conversations are retained.
  • Your API key is encrypted with DPAPI and stored locally. It is only used to authenticate with the selected provider API.

You can control the amount of data sent via the Context Depth setting (Summary or None sends less data).

For more details, see the Privacy Notice (lock icon in the header) and review the privacy policy for your selected provider.

Privacy Notice

Data Collected

  • App model context: Entity names, attribute names/types, association mappings, microflow names/parameters/activities, page names, and enumeration values from your Mendix project.
  • Chat messages: Text you type in the chat input.
  • Tool results: Data returned by tool calls (model queries, microflow operations).

How Data is Processed

  • Data is sent via HTTPS to the configured AI provider for responses.
  • The provider depends on your settings: Anthropic or OpenAI.
  • Review the privacy policy of the provider you configure before use.

Data Stored Locally

  • API key: DPAPI-encrypted in %APPDATA%\AideLite\config.json
  • Conversation history: DPAPI-encrypted in %APPDATA%\AideLite\history\ (max 50 conversations)
  • Debug log: %APPDATA%\AideLite\debug.log (rotated at 5MB, no message content)

Your Rights

  • Delete conversations: Use the History panel to delete individual conversations.
  • Export data: Use the Export button to download conversations as Markdown.
  • Control data sent: Adjust Context Depth in Settings (Summary or None sends less data).
  • Revoke access: Remove your API key from Settings at any time.

Third-Party Processing

Your configured provider processes data according to its own terms and privacy policy. If your organization is subject to GDPR or similar requirements, review the provider's data processing terms before use.
